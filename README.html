Einführung

Das Programm liest die von der Tk-Anlage über die V24-Schnittstelle ausgegebenen Gebührendaten und schreibt die aufgearbeiteten Daten in eine PostgreSQL Datenbank. Ebenso wird für jeden Datensatz auch eine Meldung via syslog() ausgegeben. Die Verbindung zur Datenbank, die sich auch auf einem anderen Rechner im Netz befinden kann, wird automatisch bei Bedarf aufgebaut und, falls eine gewisse Zeit keine Einträge mehr eingefügt wurden, auch wieder abgebaut. Falls die Datenbank genau zu dem Zeitpunkt nicht erreichbar ist, wenn Daten eingefügt werden müssen, schreibt das Programm die Daten in eine Datei. In regelmäßigen Abständen wird nun versucht, die Verbindung zum DBMS herzustellen. Sobald dies gelungen ist, werden die fehlenden Datensätze in die Tabelle geschrieben.

Zusätzliche Software

Folgende Software ist für die Benutzung von "euracom" erforderlich:

  • PostgreSQL (6.1 oder höher, es wird wohl auch mit älteren Versionen funktionieren)
  • Perl (5.003 oder höher)
  • Edmund Mergl's pgsql_perl5 (1.6) -- ist in den aktuellen PostgreSQL distributionen enthalten.

Installation

Bevor man auch nur den Versuch macht, das Programm mittels 'make' zu kompilieren, sollte man einen Blick in 'Makefile' und in 'config.h' werfen. In ersterer Datei wäre als wichtige Einstellung 'PGBASE' zu erwähnen; diese Variable muß auf das Basisverzeichnis von PostgreSQL zeigen (z.B. /usr/local/postgres). CC und CFLAGS können nach Bedarf abgewandelt werden.

Wesentlich wichtiger ist config.h! Insbesondere AREACODE, DEFAULT_DEVICE müssen auf Ihre lokale Situation angepaßt werden. (Oder sagen wir besser, wenn keine Anpassung von Nöten ist, dann sind wir Nachbarn :-) ). Es kann mit Sicherheit nichts schaden, wenn man sich alle Direktiven der Reihe mal vornimmt.

Die Portierung auf andere Unices müßte auch relativ einfach von Statten gehen, sofern das System TERMIO oder TERMIOS unterstützt. Sollte jemand Erfolg (oder Mißerfolg) bei einer Portierung haben, so bitte ich um eine kurze Nachricht, um das README (und evtl. die Sourcen) entsprechend zu updaten

Einrichten der euracom Datenbank

Zuerst ist mittels "createdb isdn" eine neue Datenbank "isdn" anzulegen. In diese Datenbank wird später eine Tabelle erzeugt, in der die Gebührendaten vom Programm abgelegt werden.

Mit make database werden alle benötigten Tabellen erzeugt, wobei für die reine Gebührenerfassung nur die Tabelle euracom von Bedeutung ist. Dem Benutzer, unter dessen Kennung "euracom" abläuft, muß (INSERT) Zugriff auf die Tabelle erlaubt werden (falls der Benutzer nicht identisch mit dem DBA ist). Evtl. muß dieser Benutzer vorher mit "createuser" eingerichtet werden. 

GRANT INSERT ON euracom TO benutzer;

Start von "euracom"

Nach der Kompilation kann man das Programm "euracom" in das gewünschte Verzeichnis moven (z.b. /usr/local/bin). Der folgende Code-Ausschnitt, den man in seine rc.local (oder entsprechendes Startupscript) übernehmen kann, startet "/var/lib/euracom/euracom" als Benutzer "phone" und leitet debugging Ausgaben des Programmes nach "/var/log/euracom.log" um. 

  echo "Starting EURACOM accounting..."
  su -f -c 'exec /var/lib/euracom/euracom /dev/ttyS2 >&/var/log/euracom.log &' phone 2>&1 1>/dev/null

Besonders zu Anfang sollte mit -p file noch ein Protokoll in file erzeugt werden. In dieses File werden sämtliche Meldungen, die von der V24- Schnittstelle kommen mitprotokolliert.

Für weitere Kommandozeilen-Optionen rufen Sie einfach euracom mit dem Switch --help auf.

Gebührenauswertung

Benötigte Tabellen

Für die Gebührenauswertung sind 2 weitere Tabellen nötig, die sich ebenfalls in der DB "isdn" befinden müssen: 

CREATE TABLE avon (
  nummer text,
  name text
);

CREATE TABLE wkn (
  nummer text,
  name text
);

Die erste Tabelle enthält das Verzeichnis der Vorwahlen ("nummer" hat die Form "+49vorwahl-ohne-null", also, z.B. "+49228" für Bonn), die zweite kann vom Benutzer mit Nummern und Namen gefüllt werden, die in der Auswertung im Klartext erscheinen sollen, z.B. ('+492364108537', 'Michael Bussmann').

Der Benutzer, der die Gebührenauswertung vornimmt, benötigt 'SELECT' Zugriff auf diese beiden Tabellen!

Als Eingabedaten für die "avon"-Tabelle empfehle ich die OKZ-Datei aus dem isdnlog Paket.

User-defined functions

Für die Auswertung ist es erforderlich, PostgreSQL um 2 Funktionen zu erweitern, nämlich prefix_match(text,text) und length(text). Kopieren Sie dazu das beim "make" erzeugte "prefixmatch.so" in ein, dem PostgreSQL- superuser zugängliches Verzeichnis (z.B. /usr/local/postgres/obj). Dann definieren Sie folgende Funktionen:

CREATE FUNCTION prefix_match (text, text) 
  RETURNS bool
  AS '/usr/local/postgres/obj/prefixmatch.so'
  LANGUAGE 'c';

CREATE FUNCTION length (text)
  RETURNS int2
  AS '/usr/local/postgres/obj/prefixmatch.so'
  LANGUAGE 'c';
